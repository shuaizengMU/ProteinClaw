Specific Aims
Protein science increasingly depends on computational analysis, including structure prediction, functional annotation, protein-protein interaction modeling, and drug target identification. The number of specialized bioinformatics tools and databases for protein research continues to grow rapidly, with hundreds of software packages and dozens of databases now available and expanding. In other areas of biology, domain-specific platforms have addressed this complexity: Seurat unifies single-cell analysis, and DNAStar integrates DNA sequence workflows, allowing researchers to discover and use tools without specialized programming skills. Protein science lacks an equivalent. While general-purpose platforms such as ToolUniverse and Biomni have emerged with broad cross-domain coverage, they lack the depth required for protein-specific research, including protein language model fine-tuning, protein-specialized analytical workflows, and domain-tailored quality control. Moreover, even when tools are available, protein researchers face persistent practical barriers: many tools involve complex hyperparameters that are well understood by developers but opaque to end users; answering a single research question often requires chaining multiple tools with manual data conversion at each step; and computational predictions must be cross-referenced against published literature and curated database annotations to be scientifically meaningful, a validation process currently performed entirely by hand. We posit that a protein-specific, AI agent-driven informatics platform, combining a deeply integrated tool and knowledge ecosystem with intelligent workflow orchestration and automated model adaptation, will significantly lower the barrier to computational protein analysis and accelerate data-driven discovery. Toward this goal, we propose to develop ProteinBox, an integrated platform unifying protein analysis tools, databases, and knowledge resources, and ProteinClaw, an AI agent-driven interface that enables natural language-guided execution of protein analysis workflows.
Aim 1: Develop ProteinBox, a unified protein bioinformatics platform with integrated tools, databases, and automated model fine-tuning. We will build ProteinBox as an open-source platform unifying ~100 protein analysis tools, ~30 databases, and knowledge retrieval services through standardized APIs and pre-configured containers. ProteinBox will leverage NLM/NCBI resources (e.g. E-utilities, BLAST, CDD, PubTator) and third-party APIs (e.g. UniProt, PDB/RCSB, InterPro, STRING), alongside containerized tools for compute-intensive tasks. Deliverables include a unified data exchange layer for cross-tool format conversion, a modular architecture for community-driven tool contributions, a local web interface for direct tool access, and an automated pLM fine-tuning module that selects optimal models and PEFT strategies from user-provided data. All software will be open-source, FAIR-compliant, and deployable locally or in cloud environments.
Aim 2: Build ProteinClaw, an AI agent system for natural language-driven protein analysis with knowledge capture, literature-augmented validation, and evolutionary workflow optimization. We will develop ProteinClaw on the OpenClaw agent framework as an intelligent interface to ProteinBox. Users describe analysis goals in natural language; the agent autonomously decomposes these into executable workflows. Key innovations: (a) a skill capture system allowing experienced researchers to encode analytical expertise (tool preferences, parameter choices, interpretation heuristics) into reusable, shareable "skills" via conversation, facilitating laboratory knowledge transfer; (b) personalized adaptation to individual users' research habits over time; (c) literature-augmented result validation that cross-references computational predictions against PubMed/bioRxiv literature, and structured predicted-vs-known comparisons, using retrieval-augmented generation (RAG) to synthesize multi-source evidence; and (d) evolutionary workflow optimization inspired by AlphaEvolve to iteratively refine tool-calling sequences and parameter configurations. ProteinClaw will support open-source, commercial, and locally deployed LLMs.
